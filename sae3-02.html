Contexte

Développer une application communicante distribuée

Ce projet consistait à concevoir un système de messagerie anonyme inspiré du principe du routage en oignon — les messages sont chiffrés en plusieurs couches successives et traversent des routeurs intermédiaires avant d'atteindre leur destinataire.

Application distribuée

Système client/serveur multi-composants avec échanges en temps réel

Sécurité & anonymisation

Routage en oignon, chiffrement RSA multi-couches, masquage d'identité

Interface & UX

Interfaces graphiques PyQt6 ergonomiques pour monitoring et communication

Architecture

Système distribué en 3 niveaux

Chaque composant joue un rôle précis dans la chaîne de communication anonymisée.

🖥️ Master (serveur central)

Gère les connexions, distribue les clés et maintient la topologie du réseau

🔀 Routeurs intermédiaires

Transfèrent les messages en supprimant une couche de chiffrement à chaque étape

👤 Clients

Permettent l'envoi et la réception de messages anonymes

Technologies clés : sockets TCP, chiffrement RSA simplifié, multi-threading pour les communications simultanées, persistance MariaDB, interfaces PyQt6.

🇬🇧 English Summary
Distributed Communication System — Onion Routing

This project involved designing a distributed messaging application based on onion routing principles. The architecture includes a central server (Master), intermediate routers, and clients. Messages are encrypted in multiple RSA layers and transmitted through several nodes to ensure anonymity. I was responsible for the graphical interfaces (PyQt6), database management (MariaDB), and project organization. This project strengthened my skills in networking, Python development, and system integration.
